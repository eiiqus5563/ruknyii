## 1. البنية العامة للمشروع

### الهيكل التقني (Monorepo)

```
Rukny.io/
├── apps/
│   ├── api/          → الخادم الخلفي (NestJS + Prisma + PostgreSQL)
│   ├── web/          → تطبيق الويب للمستخدمين (Next.js 15)
│   └── admin/        → لوحة تحكم المسؤولين (Next.js 15)
├── packages/
│   ├── database/     → مشاركة أنواع قاعدة البيانات
│   ├── types/        → الأنواع المشتركة (TypeScript)
│   └── ui/           → مكونات واجهة المستخدم المشتركة
├── nginx/            → إعدادات الخادم الوسيط
├── docker-compose.yml
└── deploy.sh
```

### التقنيات المستخدمة

| الطبقة | التقنية |
|--------|---------|
| الخادم الخلفي (API) | NestJS + TypeScript |
| قاعدة البيانات | PostgreSQL + Prisma ORM |
| التخزين المؤقت | Redis (Bull Queues + Caching) |
| تخزين الملفات | AWS S3 (متوافق) |
| تطبيق الويب | Next.js 15 + React + Tailwind CSS |
| لوحة الإدارة | Next.js 15 + React + Tailwind CSS |
| المصادقة | OAuth 2.0 (Google/LinkedIn) + JWT + 2FA |
| الإشعارات الفورية | WebSocket (Socket.IO) |
| البريد الإلكتروني | Resend API + SMTP |
| الرسائل | WhatsApp API + Telegram Bot |
| النشر | Docker + Docker Compose + Railway |

## 2. قسم المصادقة والأمان (Auth)

### الوصف
نظام مصادقة شامل يدعم تسجيل الدخول عبر OAuth، إدارة الجلسات المتعددة، والمصادقة الثنائية.

### الميزات

#### تسجيل الدخول عبر OAuth
- **Google OAuth** — تسجيل الدخول بحساب جوجل
- **LinkedIn OAuth** — تسجيل الدخول بحساب لينكد إن
- نظام تبادل الرموز (Exchange Code) للأمان الإضافي

#### إدارة الرموز (Tokens)
- **Access Token** — صالح لمدة 30 دقيقة مع ربط بمعرف الجلسة
- **Refresh Token** — صالح لمدة 14 يوم مع تشفير SHA-256
- تدوير تلقائي للـ Refresh Token عند كل تجديد
- كوكيز httpOnly مع SameSite للحماية

#### المصادقة الثنائية (2FA)
- دعم TOTP (Time-based One-Time Password)
- رموز QR لتفعيل التطبيقات مثل Google Authenticator
- رموز احتياطية (Backup Codes) في حالة فقدان الجهاز
- تشفير مفتاح 2FA بـ AES-256

#### إدارة الجلسات
- تتبع الأجهزة (المتصفح، نظام التشغيل، عنوان IP)
- بصمة الجلسة (Session Fingerprint) لاكتشاف السرقة
- عرض جميع الجلسات النشطة
- تسجيل الخروج من أجهزة محددة أو جميعها

#### حماية الأمان
- حماية من القوة الغاشمة (Brute Force):
  - تسجيل الدخول: 5 محاولات / 5 دقائق → حظر 15 دقيقة
  - إعادة التعيين: 3 محاولات / ساعة
  - OTP: 5 محاولات / 5 دقائق → حظر 30 دقيقة
  - عقوبات تصاعدية حتى 24 ساعة
- حماية CSRF مع التحقق من Origin
- QuickSign — مصادقة بدون كلمة مرور

### نقاط النهاية (Endpoints)

| الطريقة | المسار | الوصف |
|---------|--------|-------|
| POST | `/auth/google` | تسجيل الدخول بجوجل |
| POST | `/auth/linkedin` | تسجيل الدخول بلينكد إن |
| POST | `/auth/exchange-code` | تبادل الرمز المؤقت |
| POST | `/auth/update-profile` | إكمال الملف الشخصي بعد التسجيل |
| GET | `/auth/me` | بيانات المستخدم الحالي |
| POST | `/auth/refresh` | تجديد الرمز |
| DELETE | `/auth/logout` | تسجيل الخروج |

## 3. قسم المستخدمين (Users)

### الوصف
إدارة حسابات المستخدمين وإعداداتهم الشخصية والأمنية.

### الميزات

#### إدارة الملف الشخصي
- تحديث الاسم والصورة الشخصية ورقم الهاتف
- تغيير البريد الإلكتروني مع التحقق
- تخزين مؤقت للملف الشخصي (60 ثانية)

#### إعدادات الأمان
- إعداد وتفعيل/تعطيل المصادقة الثنائية
- عرض جميع الجلسات النشطة
- تسجيل الخروج من أي جهاز
- تفضيلات الإنذارات الأمنية:
  - إنذار عند تسجيل دخول من جهاز جديد
  - إنذار عند محاولات تسجيل دخول فاشلة
  - إنذار عند تغيير البريد الإلكتروني
  - إنذار عند نشاط مشبوه

#### إدارة الحساب
- تعطيل الحساب مؤقتاً (Deactivate)
- حذف الحساب نهائياً مع تنظيف البيانات

## 4. قسم الملفات الشخصية (Profiles)

### الوصف
صفحات شخصية عامة قابلة للتخصيص تعمل كصفحات هبوط (Landing Pages) للمستخدمين.

### الميزات

#### صفحة عامة بعنوان فريد
- اسم مستخدم فريد (username) كرابط عام: `rukny.io/@username`
- التحقق من توفر اسم المستخدم

#### التخصيص
- اسم العرض والسيرة الذاتية (Bio)
- صورة شخصية + صورة غلاف
- قسم Hero قابل للتخصيص مع إعدادات العلامة التجارية
- **قالبان جاهزان**: Classic و Portfolio
- مفتاح ثيم (Theme Key) للتخصيص المتقدم

#### إعدادات الخصوصية
- الرؤية: عام / خاص
- إخفاء البريد الإلكتروني
- إخفاء رقم الهاتف
- إخفاء الموقع

#### التخزين
- تتبع استخدام التخزين (Storage Usage)
- حد تخزين افتراضي: 5 جيجابايت لكل مستخدم

### نقاط النهاية

| الطريقة | المسار | الوصف |
|---------|--------|-------|
| POST | `/profiles` | إنشاء الملف الشخصي |
| GET | `/profiles/me` | الملف الشخصي الحالي |
| GET | `/profiles/check/:username` | التحقق من توفر اسم المستخدم |
| GET | `/profiles/:username` | عرض الملف الشخصي العام |
| PUT | `/profiles` | تحديث الملف الشخصي |
| POST | `/profiles/avatar` | رفع صورة الملف الشخصي (حد 5MB) |

## 5. قسم الروابط الاجتماعية (Links)

### الوصف
نظام متكامل لإدارة الروابط الاجتماعية يشبه Linktree مع تحليلات وتتبع النقرات.

### الميزات

#### إدارة الروابط
- إنشاء روابط مع عنوان ورابط مخصص
- رابط مختصر تلقائي لكل رابط
- ترتيب العرض بالسحب والإفلات
- تثبيت الروابط المهمة
- تحديد الروابط الشائعة

#### مجموعات الروابط (Link Groups)
- تنظيم الروابط في مجموعات
- لون وأيقونة مخصصة لكل مجموعة
- ترتيب المجموعات
- دعم عربي (nameAr)

#### عمليات جماعية
- إعادة ترتيب الروابط
- تفعيل/تعطيل مجموعة من الروابط
- نقل روابط متعددة إلى مجموعة
- حذف مجموعة من الروابط

#### تحليلات الروابط
- تتبع النقرات اليومي
- تحليل حسب: البلد، الجهاز، المتصفح، نظام التشغيل، المُحيل
- إحصائيات إجمالية للنقرات والمشاهدات

#### اختصار الروابط (URL Shortener)
- توليد روابط مختصرة فريدة
- تتبع التحليلات لكل رابط مختصر
- دعم تاريخ انتهاء الصلاحية

### نقاط النهاية

| الطريقة | المسار | الوصف |
|---------|--------|-------|
| POST | `/social-links` | إنشاء رابط |
| GET | `/social-links/my-links` | روابط المستخدم |
| GET | `/social-links/profile/:id` | روابط الملف الشخصي العام |
| PUT | `/social-links/:id` | تحديث رابط |
| DELETE | `/social-links/:id` | حذف رابط |
| PATCH | `/social-links/reorder` | إعادة ترتيب |

## 6. قسم النماذج (Forms)

### الوصف
نظام بناء نماذج متقدم يدعم أكثر من 25 نوع حقل، نماذج متعددة الخطوات، منطق شرطي، وتكاملات خارجية.

### الميزات

#### أنواع النماذج
| النوع | الوصف |
|-------|-------|
| CONTACT | نموذج اتصال |
| SURVEY | استبيان |
| REGISTRATION | نموذج تسجيل |
| ORDER | نموذج طلب |
| FEEDBACK | نموذج ملاحظات |
| QUIZ | اختبار |
| APPLICATION | نموذج تقديم |
| OTHER | أخرى |

#### أنواع الحقول (25+ نوع)
- **نصية**: TEXT, TEXTAREA, EMAIL, PHONE, URL, NUMBER
- **اختيارية**: SELECT, MULTI_SELECT, RADIO, CHECKBOX
- **متقدمة**: FILE, IMAGE, DATE, TIME, DATETIME, DATE_RANGE
- **تقييم**: RATING, SCALE, SLIDER, NPS
- **خاصة**: MATRIX, SIGNATURE, RECAPTCHA, HIDDEN, HEADING, PARAGRAPH, DIVIDER
- **موقع**: LOCATION, ADDRESS, COUNTRY

#### نماذج متعددة الخطوات (Multi-Step)
- تقسيم النموذج لخطوات
- شريط تقدم
- ترقيم الأسئلة
- خلط ترتيب الأسئلة

#### المنطق الشرطي (Conditional Logic)
- إظهار/إخفاء الحقول بناءً على إجابات سابقة
- شروط متعددة مع AND/OR

#### إعدادات النموذج
- السماح بإجابات متعددة أو إجابة واحدة لكل مستخدم
- حد أقصى للإرسالات
- تاريخ فتح وإغلاق
- رد تلقائي على المرسل
- إشعار البريد عند كل إرسال
- Webhook لإرسال البيانات لخدمات خارجية

#### التصميم والمظهر
- صورة غلاف مخصصة (WebP مع BlurHash)
- صور بانر متعددة (وضع عرض: فردي/شريحة)
- ثيم مخصص (JSON) لألوان الخطوط والخلفيات

#### التكاملات
- **Google Sheets** — تصدير الإجابات تلقائياً
- **Google Drive** — حفظ الملفات المرفوعة
- **Webhook** — إرسال البيانات لأي خدمة خارجية

#### تحليلات النماذج
- عدد المشاهدات والإرسالات
- معدل الإكمال
- متوسط وقت الإكمال
- تحليلات حسب الجهاز والمتصفح
- تحليلات جغرافية (بلد، مدينة)
- تحليلات لكل حقل (عدد الإجابات، التخطي، توزيع القيم)

### نقاط النهاية

| الطريقة | المسار | الوصف |
|---------|--------|-------|
| POST | `/forms` | إنشاء نموذج |
| GET | `/forms` | نماذج المستخدم (مع تصفية) |
| GET | `/forms/public/user/:username` | نماذج عامة بالـ username |
| GET | `/forms/public/:slug` | عرض نموذج عام |
| POST | `/forms/public/:slug/submit` | إرسال نموذج (حد: 10/دقيقة) |
| PUT | `/forms/:id` | تحديث نموذج |
| DELETE | `/forms/:id` | حذف نموذج |
| GET | `/forms/:id/responses` | عرض الإجابات |
| POST | `/forms/:id/export` | تصدير إلى Google Sheets |
| POST | `/forms/upload` | رفع صورة الغلاف |

## 7. قسم الأحداث (Events)

### الوصف
نظام إدارة أحداث متكامل يدعم الأحداث الحضورية والافتراضية مع التسجيل، التذاكر، قوائم الانتظار، والتقييمات.

### الميزات

#### أنواع الأحداث (19 نوع)
CONFERENCE، WORKSHOP، WEBINAR، HACKATHON، SPORTS، MEETUP، NETWORKING، SEMINAR، TRAINING، EXPO، CONCERT، FESTIVAL، CHARITY، COMPETITION، PANEL، ROUNDTABLE، BOOTCAMP، RETREAT، OTHER

#### حالات الأحداث
| الحالة | الوصف |
|--------|-------|
| SCHEDULED | مجدول |
| ONGOING | جاري |
| COMPLETED | مكتمل |
| CANCELLED | ملغي |

#### إنشاء الأحداث
- عنوان ووصف مع رابط slug فريد (6 أحرف)
- تاريخ البداية والنهاية
- الموقع والمكان (حضوري/افتراضي)
- الحد الأقصى للحضور
- السعر (مجاني/مدفوع)
- صورة الحدث
- رابط الاجتماع الافتراضي + كلمة المرور
- تصنيف الحدث

#### نظام التسجيل
- حالات التسجيل: PENDING, CONFIRMED, CANCELLED, ATTENDED, NO_SHOW
- عدد الحضور لكل تسجيل
- ملاحظات المسجل
- تواريخ: التسجيل، التأكيد، الإلغاء

#### نظام قائمة الانتظار (Waitlist)
- ترتيب تلقائي في قائمة الانتظار عند امتلاء الحدث
- حالات: WAITING, NOTIFIED, ACCEPTED, EXPIRED, CANCELLED
- إشعار تلقائي عند توفر مكان
- مدة انتهاء الإشعار

#### نظام التذاكر
- رقم تذكرة فريد
- رمز QR فريد لكل تذكرة
- حالات: VALID, USED, CANCELLED, EXPIRED
- تسجيل الحضور (Check-in) مع تتبع من قام بذلك

#### منظمو الأحداث (Multi-Organizer)
- أدوار: OWNER, CO_ORGANIZER, MODERATOR, ASSISTANT
- دعوات مع حالات: PENDING, ACCEPTED, DECLINED, EXPIRED, CANCELLED
- صلاحيات مخصصة لكل منظم

#### رعاة الأحداث
- مستويات الرعاية: PLATINUM, GOLD, SILVER, BRONZE, PARTNER
- شعار، موقع، ووصف الراعي
- ترتيب العرض

#### تقييمات الأحداث
- تقييم بالنجوم + تعليق
- تقييم مجهول (اختياري)
- تقييم واحد لكل مستخدم لكل حدث

#### تحديثات فورية (WebSocket)
- إشعار بتسجيل جديد
- إشعار بإلغاء تسجيل
- تحديث عدد الحضور
- ترقية من قائمة الانتظار
- تغيير حالة الحدث
- إعلانات المنظم

#### تكامل Google Calendar
- مزامنة الأحداث مع تقويم جوجل
- معرف تقويم جوجل + رابط Google Meet

### نقاط النهاية

|---------|--------|-------|

| GET | `/events` | الأحداث العامة (مع تصفية) |
| GET | `/events/my-events` | أحداث المستخدم |
| GET | `/events/my-registrations` | تسجيلات المستخدم |
| GET | `/events/e/:slug` | حدث برابط slug |
| GET | `/events/:id` | تفاصيل الحدث |

## 8. قسم المتاجر والتجارة الإلكترونية (Stores)

### الوصف
منصة تجارة إلكترونية متكاملة تتيح لكل مستخدم إنشاء متجر خاص مع منتجات، طلبات، عربة تسوق، كوبونات، وتتبع الشحنات.

### 8.1 إدارة المتاجر

#### الميزات
- متجر واحد لكل مستخدم
- رابط فريد (slug) للمتجر
- تصنيفات المتاجر (20 تصنيف):
  أزياء، إلكترونيات، طعام، تجميل، منزل، رياضة، كتب، مجوهرات، صحة، تصوير، سيارات، سفر، هدايا، حيوانات أليفة، أطفال، خدمات، يدوي، عضوي، مقهى، أخرى
- شعار وبانر للمتجر
- معلومات الاتصال والموقع
- عدد الموظفين (فردي، 2-5، 6-10، 11-50، +50)
- إحصائيات المتجر مع تخزين مؤقت 5 دقائق

### 8.2 إدارة المنتجات

#### الميزات
- إنشاء منتجات مع وصف عربي وإنجليزي
- **التسعير**: سعر أساسي + سعر تخفيض + عملة (IQD افتراضياً)
- **المخزون**: تتبع الكميات مع دعم SKU
- **التصنيف**: تصنيفات خاصة بكل متجر

- صور الملف الشخصي: 400x400px
- صورة الغلاف: 1400x400px
- غلاف النموذج: 1200x630px

## 14. البنية التحتية (Infrastructure)

### 14.1 المراقبة (Monitoring)
- فحوصات صحة شاملة: قاعدة البيانات، Redis، S3
- تتبع استخدام الذاكرة ووقت التشغيل
- مقاييس Prometheus (عدادات، مقاييس، مدرجات)
- مستويات الصحة: healthy, degraded, unhealthy
- نقطة `/healthz` سريعة لـ Load Balancer

### 14.2 طوابير المعالجة (Queue - Bull/Redis)
4 طوابير متخصصة:

| الطابور | المهام |
|---------|--------|
| Email | ترحيب، تحقق، إنذارات أمنية |
| Image | تحسين، تغيير حجم، تحويل صيغة |
| Notification | Push notifications، إشعارات داخلية |
| Cleanup | تنظيف مجدول، حذف ملفات مؤقتة |

- إعادة المحاولة مع تأخير تصاعدي
- تنظيف تلقائي للمهام المكتملة/الفاشلة
- دعم الأولوية

### 14.3 تحديد المعدل (Rate Limiting)
خطط متدرجة:

| الخطة | الحد / دقيقة |
|-------|---------------|
| Anonymous | 100 |
| Free | 200 |
| Premium | 2000 |
| Enterprise | 10000 |

- خوارزمية النافذة المنزلقة (Sliding Window)
- حدود خاصة لنقاط حساسة (/auth/login: 5/5min)
- ترويسات Retry-After

### 14.4 الأمان (Security)
- **كشف الشذوذ**: تغيير الموقع (حد 2 ساعة)، تغيير IP (5 مختلفة/ساعة)، تغيير الجهاز (3 أجهزة/يوم)
- **بصمة الجهاز**: تتبع الأجهزة الموثوقة
- **سجل الأمان**: تخزين مركزي مع تحديثات WebSocket فورية
- **reCAPTCHA Enterprise**: تحدي CAPTCHA للنشاط المشبوه
- **تصدير التدقيق**: تقارير أمنية بصيغة PDF

### 14.5 التخزين المؤقت (Cache - Redis)
- **إعادة الاتصال**: Exponential backoff تلقائي
- **التغليف (Wrap)**: TTL + حساب + مقاييس
- **إبطال بالعلامات**: لتجميع التخزين المؤقت
- **المقاييس**: نسبة الإصابة/الخطأ، زمن الاستجابة
- استراتيجيات TTL: ملف شخصي (5 دقائق)، أحداث (2 دقيقة)، بحث (30 ثانية)

### 14.6 قاعدة البيانات
- **مجمع الاتصالات**: 15 اتصال (إنتاج)، 10 (تطوير)
- **مهلات الاستعلام**: 
  - افتراضي: 10 ثوانٍ
  - سريع: 5 ثوانٍ
  - معقد: 20 ثانية
  - تقارير: 60 ثانية
  - صيانة: 5 دقائق
- **تنظيف مجدول**: كل ساعة عبر Cron مع قفل موزع
- **حذف مجمع**: 1000 سجل في الدفعة

### 14.7 التحقق من البيئة (Config)
- تحقق صارم عند بدء التشغيل عبر Zod
- DATABASE_URL (صيغة URL)
- JWT_SECRET (32 حرف كحد أدنى)
- TWO_FACTOR_ENCRYPTION_KEY (32 حرف كحد أدنى)
- فشل سريع إذا كانت الإعدادات غير صالحة

## 18. المقترحات والتحسينات المستقبلية

### 🔴 تحسينات ذات أولوية عالية

#### 1. نظام الدفع الإلكتروني
- تكامل مع بوابات دفع عراقية (ZainCash, FastPay, NassPay)
- تكامل Stripe/PayPal للدفع الدولي
- محفظة إلكترونية داخلية
- فواتير تلقائية بصيغة PDF

#### 2. تطبيق موبايل (React Native / Flutter)
- تطبيق أصلي لنظامي iOS و Android
- إشعارات Push أصلية
- كاميرا متكاملة لتصوير المنتجات
- ماسح QR للتذاكر

#### 3. نظام الدردشة المباشرة
- محادثات فورية بين البائع والمشتري
- دعم الوسائط المتعددة (صور، مقاطع صوتية)
- حالة "متصل/غير متصل"
- أرشيف المحادثات

#### 4. تحسين محركات البحث (SEO)
- بيانات Schema.org المنظمة لكل صفحة
- خريطة موقع ديناميكية (Sitemap)
- Open Graph محسّن للمشاركة الاجتماعية
- AMP للمنتجات والأحداث

### 🟡 تحسينات ذات أولوية متوسطة

#### 5. نظام العمولات والشراكات (Affiliate)
- روابط إحالة فريدة
- تتبع التحويلات
- لوحة شراكات للمسوقين
- نظام عمولات مرن (نسبة/ثابت)

#### 6. الذكاء الاصطناعي
- اقتراحات أوصاف المنتجات بالذكاء الاصطناعي
- تصنيف تلقائي للمنتجات
- إنشاء نماذج ذكية من وصف نصي
- Chatbot ذكي لخدمة العملاء
- توصيات منتجات مخصصة

#### 7. نظام اشتراكات متقدم
- خطط مدفوعة (Basic, Premium, Enterprise)
- ميزات حصرية لكل خطة
- فترة تجربة مجانية
- فوترة شهرية/سنوية

#### 8. تقارير ولوحات معلومات متقدمة
- تقارير المبيعات التفصيلية (يومي/أسبوعي/شهري/سنوي)
- تقارير سلوك العملاء
- تقارير أداء المنتجات
- تصدير التقارير (PDF, Excel, CSV)
- رسوم بيانية تفاعلية متقدمة

#### 9. نظام شحن متكامل
- ربط مع شركات شحن عراقية
- حساب تكلفة الشحن تلقائياً حسب الموقع
- تتبع الشحنات في الوقت الفعلي
- طباعة بوليصة الشحن

#### 10. نظام نقاط الولاء
- نقاط مكافآت على كل عملية شراء
- استبدال النقاط بخصومات
- مستويات الولاء (برونزي، فضي، ذهبي، بلاتيني)

### 🟢 تحسينات ذات أولوية منخفضة

#### 11. سوق متعدد البائعين (Marketplace)
- صفحة استكشاف للمتاجر
- بحث وتصفية عبر جميع المتاجر
- مقارنة المنتجات
- تقييم المتاجر

#### 12. نظام المدونات
- مدونة لكل مستخدم/متجر
- محرر نصوص غني (Rich Text Editor)
- تصنيف المقالات
- SEO للمدونة

#### 13. قوالب متاجر جاهزة
- قوالب احترافية متعددة
- تخصيص الألوان والخطوط
- سحب وإفلات لتخصيص الأقسام
- معاينة حية

#### 14. API عام للمطورين
- توثيق Swagger/OpenAPI كامل (موجود جزئياً)
- مفاتيح API للمطورين
- Webhooks للأحداث
- SDK للغات البرمجة الشائعة

#### 15. تحسينات إضافية
- **دعم متعدد اللغات كامل**: واجهة + محتوى بأكثر من لغة
- **نظام مراسلة داخلي**: رسائل بين المستخدمين
- **نظام حجز المواعيد**: Calendly-like مدمج
- **تصدير/استيراد المنتجات**: CSV/Excel
- **إدارة المخزون المتقدمة**: إنذارات نفاد المخزون، تاريخ المخزون
- **نظام إرجاع واسترداد**: سياسة الإرجاع، طلبات الاسترداد
- **تقييم NPS للمنصة**: رضا العملاء عن المنصة ككل
